🛡 Remote Access Audit

Technician Quick Sheet — scan a PC for unauthorized remote-access software
Nerdy Neighbor
audit.nerdyneighbor.net

Run it

On the customer's PC, click Start, type PowerShell, open Windows PowerShell, then paste this line and press Enter:
irm audit.nerdyneighbor.net | iex
Click Yes on the blue User Account Control (UAC) prompt that appears. That's the only click needed to start.

What happens

  1. Downloads the latest version automatically (nothing to install or update).
  2. Asks for Administrator rights once (UAC → Yes).
  3. Runs all scans (1–2 min; longer on big drives — let it finish).
  4. Opens the remediation window with every finding.
  5. Saves an HTML report to the Desktop and opens it when you close the window.

What it checks

Processesrunning remote-access / unsigned tools
Servicesremote-control services & bad paths
Networkconnections on remote/C2 ports
Tasksscheduled tasks launching RATs
Registrystartup (Run) entries
FoldersTeamViewer, AnyDesk, VNC, etc.
RDPRemote Desktop on/off + security
ScreenConnectincl. hidden LSA-DLL persistence
AppDatatools buried deep in user folders
Usersadmin & Remote Desktop accounts
Trafficsuspicious outbound / beaconing

Reading the results

Each finding has a severity:
  • HIGH  Known remote tool or unsigned program in a hidden spot — investigate now.
  • MEDIUM  Suspicious but needs context (e.g. odd location, management port).
  • LOW  Worth a look — e.g. an admin account, or unsigned exe buried in AppData.

Using the window

  • ▶ View fix script — see exactly what a fix will do before running it.
  • Fix button (e.g. Kill Process, Full Removal) — runs that one fix in an Admin window; confirm when asked.
  • 📋 View Script — opens all fixes in Notepad to copy/save.
  • ⚡ Run ALL Fixes — runs every fix in turn (use carefully; each asks to confirm).
If a remote-access tool is confirmed and unauthorized:
  1. Disconnect the PC from the internet.
  2. Run the matching Fix / removal steps.
  3. Have the customer change passwords from a different device.
  4. Reboot if prompted (ScreenConnect DLL removal needs it).
Notes. Legitimate remote tools are reported too — confirm each one is authorized before removing. Always-on report copy is on the Desktop: RemoteAccessAudit_<date>.html. If irm errors on a very old PC, run [Net.ServicePointManager]::SecurityProtocol=[Net.SecurityProtocolType]::Tls12 first, then paste the command again.
